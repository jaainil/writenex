# Plan: Keystatic-style Fields API for @writenex/astro

## Context

The `@writenex/astro` package currently has a basic schema system with only 7 field types (`string`, `number`, `boolean`, `date`, `array`, `image`, `object`) and no builder API. The goal is to add a comprehensive Fields API inspired by Keystatic with 25+ field types, a TypeScript-first builder pattern (`fields.text()`, `fields.select()`, etc.), and updated form rendering components.

## API Design

Use a **builder function pattern** (like Keystatic) where each field type is a method on a `fields` object that returns a `FieldDefinition`:

```typescript
import { fields, collection } from '@writenex/astro/config';

const myCollection = collection({
  name: 'blog',
  path: 'src/content/blog',
  schema: {
    title: fields.text({ label: 'Title', validation: { isRequired: true } }),
    slug: fields.slug({ name: { label: 'Slug' } }),
    status: fields.select({ options: ['draft', 'published'], defaultValue: 'draft' }),
    heroImage: fields.image({ directory: 'public/images/blog' }),
    tags: fields.array(fields.text({ label: 'Tag' })),
    body: fields.markdoc({ label: 'Content' }),
  }
});
```

## Files to Create

### 1. `packages/astro/src/fields/types.ts` — Core field type definitions

Define the `FieldDefinition` base interface and per-field-type configs:

```typescript
// Base field definition returned by every builder function
interface FieldDefinition {
  fieldKind: 'text' | 'slug' | 'url' | 'number' | 'integer' | 'select' | 'multiselect'
           | 'checkbox' | 'date' | 'datetime' | 'image' | 'file' | 'object'
           | 'array' | 'blocks' | 'relationship' | 'path-reference'
           | 'markdoc' | 'mdx' | 'conditional' | 'child' | 'cloud-image'
           | 'empty' | 'empty-content' | 'empty-document' | 'ignored';
  label?: string;
  description?: string;
  validation?: ValidationOptions;
  defaultValue?: unknown;
}

interface ValidationOptions {
  isRequired?: boolean;
  min?: number;
  max?: number;
  minLength?: number;
  maxLength?: number;
  pattern?: string;
  patternDescription?: string;
}
```

Per-field config interfaces (e.g., `TextFieldConfig`, `SelectFieldConfig`, `ImageFieldConfig`, etc.)

Converts `FieldDefinition` → the internal `SchemaField` type used by the form system. Maps `fieldKind` to the appropriate `FieldType` string.

### 4. `packages/astro/src/fields/index.ts` — Barrel exports

Exports `fields`, `collection`, `singleton`, and all field types.

## Files to Modify

### 1. `packages/astro/src/types/config.ts`

- Expand `FieldType` union to include all 25+ types
- Expand `SchemaField` interface to support:
  - `label?: string` — display label
  - `options?: string[]` — for select/multiselect
  - `directory?: string` — for image/file storage
  - `publicPath?: string` — for image/file URLs
  - `validation?: ValidationOptions` — validation rules
  - `fields?: Record<string, SchemaField>` — for object type (nested fields)
  - `itemField?: SchemaField` — for array type (item schema)
  - `blockTypes?: Record<string, SchemaField>` — for blocks type
  - `collection?: string` — for relationship type
  - `multiline?: boolean` — for text type
  - `format?: string` — for date/datetime type
  - `itemLabel?: string` — for array/blocks item labels
  - `matchField?: string` — for conditional type

## Implementation Order

1. **Types first** — Create `fields/types.ts` with all field definition interfaces
2. **Builder functions** — Create `fields/fields.ts` with `fields` object
3. **Collection/Singleton helpers** — Create `fields/collection.ts`
4. **Resolve layer** — Create `fields/resolve.ts` for FieldDefinition → SchemaField conversion
5. **Update config types** — Expand `SchemaField` and `FieldType` in `types/config.ts`
6. **Update Zod schemas** — Expand `config/schema.ts` validation
7. **Update exports** — Wire up all new exports
8. **Update form rendering** — Add all new field renderers to `FrontmatterForm.tsx`
9. **Update schema detection** — Enhance `discovery/schema.ts`
